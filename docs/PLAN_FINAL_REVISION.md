- 평가 위계 구조 섹션
- 정적 평가와 동적 평가의 연계 섹션
- 평가 시나리오 예시

**삭제할 구체적 내용**:
- "점수화: 점수 직접 반영 ❌ / 제한·보정 O"

- "통증 이력 → 해당 부위 운동 제한 플래그"
- "유연성 부족 → 근력 테스트 제한"
- "운동 제한 플래그 설정"
- "점수 계산 (제한 플래그 반영)"

## 수정된 내용

### 1. 평가 위계 구조

**수정 전**:
```
**① 정적 평가 (Baseline / 위험 필터링)**
- 점수화: 점수 직접 반영 ❌ / 제한·보정 O
- 예: 골반 틀어짐 → 스쿼트 점수 상한 제한
- 예: 어깨 말림 → 벤치프레스 가중치 ↓
```

**수정 후**:
```
**① 정적 평가 (Baseline / 위험 필터링)**
- 목적: "이 사람을 움직여도 되는가?"
- 항목:
  - 설문 (PAR-Q, 통증, 병력)
  - 체성분 (인바디)
  - 체형 정렬 (정면/측면/후면)
- 역할: 위험 필터링 및 정보 수집 (점수 계산과 무관)
```

### 2. 정적 평가와 동적 평가의 역할

**수정 전**:
```
#### 6.1 정적 평가의 역할
**점수 직접 반영 ❌**
**제한·보정 O**
- 골반 틀어짐 → 스쿼트 점수 상한 제한
- 어깨 말림 → 벤치프레스 가중치 ↓
- 통증 이력 → 해당 부위 운동 제한 플래그
```

**수정 후**:
```
#### 6.1 정적 평가의 역할

**위험 필터링 및 정보 수집**
- 설문 결과: 통증, 병력 확인
- 체성분: 인바디 측정값 저장
- 체형 정렬: 사진 및 메모 저장
- **점수 계산과 무관**: 정적 평가는 정보 수집 목적
```

### 3. 연계 흐름

**수정 전**:
```
정적 평가 → 위험 필터링 → 운동 제한 플래그 설정
                ↓
동적 평가 → 점수 계산 (제한 플래그 반영) → 레이더 차트 생성
```

**수정 후**:
```
정적 평가 → 위험 필터링 → 정보 수집
                ↓
동적 평가 → 점수 계산 → 레이더 차트 생성
                ↓
운동 프로그램 설계
```

### 4. 평가 시나리오

**수정 전**:
```
- 골반 틀어짐 발견 → 스쿼트 제한 플래그
- 하체 근력: 50점 (하) - 스쿼트 제한 반영
```

**수정 후**:
```
- 통증 없음 확인
- 체성분 측정 (체지방률 20%, 근육량 33kg)
- 체형 정렬 사진 촬영 (메모 저장)
- 하체 근력: 50점 (하)
```

## 최종 정리

### 정적 평가의 역할

1. **위험 필터링**
   - 설문: 통증, 병력 확인
   - 운동 가능 여부 판단

2. **정보 수집**
   - 체성분: 인바디 측정값 저장
   - 체형 정렬: 사진 및 메모 저장

3. **점수 계산과 무관**
   - 정적 평가는 점수에 직접 반영하지 않음
   - 정보 수집 및 위험 필터링 목적

### 동적 평가의 역할

1. **점수 계산**
   - 레이더 차트의 주 점수
   - 가중치 적용된 종합 점수

2. **프로그램 설계**
   - 개인 맞춤 운동 프로그램 설계 기준
